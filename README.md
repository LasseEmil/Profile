# Profile Page

Personal blog/portfolio website for GitHub Pages.

## Current Structure

This iteration now includes separate pages stored under `Source/Website/`:
- `Source/Website/index.html` (home)
- `Source/Website/about.html`
- `Source/Website/research.html`
- `Source/Website/work.html`
- `Source/Website/blog.html`
- `Source/Website/contact.html`

All pages share a common top navigation and stylesheet (`Source/Website/styles.css`).

Supporting directories:
- `Source/Updates/` – Markdown log plus converter script for personal updates.
- `Stage/Build/` – build scripts (run `python Stage/Build/stage.py`).
- `Stage/Builds/` – git-ignored build outputs generated by `stage.py`.

### Personal updates

- Log dated entries in `Source/Updates/Updates.md`.
- Run `python Source/Updates/build_updates.py` (the lightweight “farmdown” converter) to generate `Source/Website/updates.html`.
- The homepage automatically fetches `updates.html` and displays the latest items in the “Latest Updates” section.

Example entry:

```
## 2026-03-01 – Shipped Dark Mode
- Added a dark theme toggle across all pages.
- Wrote a quick note about our theming approach.
```


## Deployment

GitHub Pages is deployed via `.github/workflows/pages.yml`, which runs `python Stage/Build/stage.py --name github-pages` to render updates and copy the site into `Stage/Builds/github-pages/` before uploading to Pages with a `.nojekyll` marker.

### Staging builds locally

```
python Stage/Build/stage.py --name local
```

The command above refreshes `Source/Website/updates.html` and writes a complete static build to `Stage/Builds/local/`. Inspect that folder (or serve it with a local HTTP server) before committing. Create additional builds by changing `--name`; the Pages workflow uses the `github-pages` build name automatically.
